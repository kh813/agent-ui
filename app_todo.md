# 実装タスクリスト：Universal AI Desktop

本ドキュメントは、Rust (Tauri v2) を使用した「Universal AI Desktop」開発における実装手順と開発方針を定義します。

**【v2 更新概要】**
- Phase 2: Ptyプロセス起動時に作業ディレクトリ（cwd）を指定できるようタスクを追加。
- Phase 4: 作業ディレクトリ表示・変更バーのUI実装タスクを追加。
- Phase 5: 旧タスク5-7（バイナリ検出）を新Phase 6に移設し、認証系タスクに集中させた。
- **Phase 6（新設）**: 「AIエージェントの検出・初回インストール導線・作業ディレクトリ管理」を新設。CLIバイナリを同梱できない問題への対応と、作業ディレクトリ機能のバックエンド結合をここに集約。
- 旧Phase 6「統合テスト・パッケージング」はPhase 7に繰り下げ、CLI名称をAntigravity CLIに更新。

**【v3 更新概要】**
- Phase 6: 「初回起動時のみ」のインストール導線から、「初回起動時の対話的確認」＋「設定画面からいつでも追加インストール可能」な導線に拡充。
- **Phase 7（新設）**: 「アップデート管理機能」を新設。`agy`/`claude`/`codex`でアップデート方式が異なることに対応するタスクを集約。
- 旧Phase 7「統合テスト・パッケージング」はPhase 8に繰り下げ。

**【v4 更新概要】**
- **Claude Codeのサポート除外**: Claude Codeは有償契約時にClaude Desktop経由でGUI実行できるため、初期サポート対象から完全に除外し、将来対応（またはサポート外）に変更。
- 初期サポート対象を **Antigravity CLI (`agy`) のみ**に限定し、ToDoリストから `claude` (Claude Code) を除外または将来対応用の設計タスクに縮小。

## 開発方針

各フェーズの開発を進めるにあたり、以下のルールを厳守します。

1. **フェーズ分割の厳守**
   * 開発は以下のフェーズ順に進めます。
   * 1フェーズ内のタスク数は10個以下とします。
2. **フェーズ完了の定義とGitコミット**
   * フェーズ内の全タスク実装後、以下のプロセスを踏んでからフェーズ完了とします。
     1. **テストコードの実行**: 各フェーズで実装した機能に対する単体テスト・結合テストを実行し、すべてパスすることを確認する。
     2. **テストビルド**: `cargo tauri build` (または開発ビルド `npm run tauri dev`) を実行し、コンパイルエラーがないことを確認する。
     3. **Gitコミット**: テストとビルドが成功した段階で、必ず `git commit` を実行してコードを確定させる。
3. **テスト駆動の意識**
   * ロジック（Pty制御、ストリームパース、認証・プロンプト検知、CLI検出・インストール実行、作業ディレクトリ管理等）を実装する際は、なるべくテストコード（Rustの `#[cfg(test)]` またはTypeScriptのテスト）を書き、自動テストで品質を担保します。
4. **CI/CD（GitHub Actions）の実行制限とローカル事前検証**
   * GitHub Actionsの実行回数制限に対応するため、明示的なリリース指示（バージョンタグの付与やバージョン更新時）以外はGitHub Actionsを動作させない設定とします。
   * GitHub Actions実行（リリース）の指示を出す場合は、事前にローカルでActionsと同じ検証（テストコードの実行、リリースビルド）を完全に通し、エラーが起きないことをローカルテストで実証してから `git commit` およびタグプッシュを行います。
5. **外部CLI仕様への依存に関する注意（新設）**
   * 初期サポートの `agy`（Antigravity CLI）および将来対応の外部CLIは本アプリの管理下にない外部ツールであり、インストールコマンド・出力フォーマット・認証フローは各社の都合で変更されます。Phase 6のタスクに着手する前に、必ず公式ドキュメントで最新の仕様を再確認してください。ソースコードに直接ハードコードせず、設定ファイル（`install_commands.json`等）に分離し、最終確認日を記載します。
6. **テストおよび実行ハングアップ（デッドロック・無限ループ）回避方針（新設）**
   * **PTYバッファ詰まり防止**: PTYプロセスの終了待機（`child.wait()` 等）を呼び出す前に、必ず出力ストリームの読み込みバッファを消費（読み切り）させてください。これを怠るとOSのパイプ容量上限に達した段階でプロセスが完全にフリーズ（ハング）します。
   * **JavaScript正規表現の無限ループ回避**: Javascript/TypeScriptでバックスペース文字（`\x08`）や制御コードを置換・検索する際、正規表現内で `\b` をブラケット外で用いると「単語境界」と誤認され、文字が置換されずに `while` ループが無限ループに陥ります。バックスペースの検出・置換には必ず `\x08` (16進) または `[\b]` を明示して使用してください。
   * **Vitestマルチスレッド競合の回避**: 制限のあるサンドボックス環境下では、Vitest のマルチスレッドプール（Worker Threads）の初期化でプロセスがハングアップすることがあります。高速かつ安定したフロントエンドテストを実行するため、Node.js標準の `assert` と `npx tsx` を用いた直接実行アプローチを優先的に採用してください。
